# SOOMPYO 생성 AI 도구 및 파일명 규칙

## 문서 범위

이 문서는 이슈 #16의 범위에 맞춰 숨표 / SOOMPYO 광고 제작에 사용할 생성 AI 도구, 대체 도구, 크레딧 대응 전략, 파일명 규칙을 확정한다.

- 포함: 이미지, 비디오/변환, 오디오 생성 도구와 대체 도구, 선택 이유, 사용 목적, 파일명 규칙
- 제외: 실제 이미지 생성, 실제 영상 생성 또는 변환, 실제 오디오 생성, 스토리보드 PDF 내보내기, 최종 MP4 편집
- 기준 문서: `MISSION.md`, `BRAND.md`, `STORYBOARD.md`

## 도구 선정 원칙

- 접근성 우선: 학습자가 비교적 쉽게 접근할 수 있는 도구를 먼저 사용한다.
- 이미지 선행 확정: 크레딧 소모가 큰 비디오 생성 전에 이미지 단계에서 구도와 스타일을 확정하고, 모든 비디오 생성/변환은 직전 단계의 확정 이미지 또는 키비주얼을 입력으로 사용한다.
- 브랜드 일관성 유지: 아이보리, 딥틸, 유자색 포인트, 슬림 캔, 쉼표 모양의 숨결 모티프를 모든 시각 생성 프롬프트의 공통 기준으로 둔다.
- 통합 편집 최소화: 편집 도구는 컷 편집, 자막, 간단한 색보정, 오디오 레벨 조정 용도로만 사용한다.

## 크레딧 및 접근성 대응 전략

- 이미지 우선 확정: 각 씬의 구도, 색감, 제품 형태를 이미지 단계에서 먼저 확정한 뒤 비디오 변환을 진행한다. 모든 영상 컷은 해당 씬의 확정 이미지 산출물을 입력 또는 기준으로 생성한다.
- 축소 제작안: 크레딧이 부족하면 6씬 38초 구성을 4씬 32초 구성으로 줄인다.
  - 축소 씬 1: 문제 제시
  - 축소 씬 2: SOOMPYO 전환
  - 축소 씬 3: 한 모금과 감각 변화
  - 축소 씬 4: 제품, 브랜드명, CTA
- 모션 대체: 비디오 생성 대기열이나 플랜 제한이 있으면 정지 이미지에 패닝, 줌, 루프 조명 변화 중심의 짧은 모션을 적용한다.
- 스타일 고정: 모든 시각 프롬프트에 `urban calm wellness drink`, `ivory background`, `deep teal shadow`, `yuja yellow accent`, `slim can`, `comma-shaped breath symbol`을 공통 스타일 키워드로 유지한다.
- 오디오 대체: Suno 사용이 어려우면 ElevenLabs로 내레이션 또는 짧은 사운드 요소를 만들고, 배경음악은 생성 가능한 범위로 축소한다.

## 파일명 규칙

### 기본 형식

```text
soompyo_s{NN}_{asset}_{variant}.{ext}
```

- `soompyo`: 브랜드 식별자
- `s{NN}`: 두 자리 씬 번호, 예: `s01`, `s02`, `s06`
- `{asset}`: 에셋 종류
- `{variant}`: 버전 또는 용도, 예: `v01`, `v02`, `final`
- `{ext}`: 파일 확장자

### 에셋 종류

| 에셋 종류 | 의미 | 예시 확장자 |
| --- | --- | --- |
| `keyvisual` | 이미지 생성 결과 또는 씬 대표 이미지 | `.png`, `.jpg` |
| `motion` | 같은 씬의 `keyvisual`을 입력 또는 기준으로 만든 비디오 생성/변환 결과 | `.mp4`, `.mov` |
| `bgm` | 배경음악 | `.wav`, `.mp3` |
| `sfx` | 효과음 | `.wav`, `.mp3` |
| `voice` | 내레이션 또는 음성 합성 | `.wav`, `.mp3` |
| `prompt` | 프롬프트 원문 또는 수정 기록 | `.md`, `.txt` |

### 파일명 예시

- `soompyo_s01_keyvisual_v01.png`
- `soompyo_s03_motion_v01.mp4`
- `soompyo_s04_bgm_v01.wav`
- `soompyo_s04_sfx_can-open_v01.wav`
- `soompyo_s06_keyvisual_final.png`
- `soompyo_s02_prompt_v02.md`
- `soompyo_final_16x9_v01.mp4`

## 검증 메모

- 이미지 생성 도구 1종 이상과 대체 도구 1종 이상을 정했다.
- 비디오 생성 또는 변환 도구 1종 이상과 대체 도구 1종 이상을 정했다.
- 오디오 생성 도구 1종 이상과 대체 도구 1종 이상을 정했다.
- 각 도구의 사용 목적과 선택 이유를 `MISSION.md`의 멀티모달 생성 도구 요구와 연결했다.
- 모든 비디오 생성/변환 단계가 같은 씬의 이전 이미지 산출물(`keyvisual`)을 활용하도록 정했다.
- 크레딧 부족, 대기열, 플랜 제한에 대비한 범위 조정 전략을 정했다.
- 파일명 규칙은 씬 번호와 에셋 종류를 일관되게 표현하며, 예시 파일명끼리 충돌하지 않는다.
