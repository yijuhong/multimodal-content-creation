# AI로 글·그림·영상 만들어 나만의 광고 만들기

| 항목 | 내용 |
| --- | --- |
| 분야 | AI 도구 학습 |
| 구분 | AI 도구 |
| 학습시간 | 40시간 |
| 문제기술 | 기술적 설명 |

## 1. 미션 소개

이제 텍스트로 이미지를 한 장 만드는 것은 누구나 하는 일이 되었습니다. 하지만 그 이미지들에 생명력을 불어넣어 하나의 '완성된 브랜드 경험'을 만드는 것은 차원이 다른 문제입니다.

광고, 티저 영상, 숏폼 캠페인처럼 시각과 청각을 동시에 사로잡는 몰입감은 어떻게 설계되는 걸까요? 단순히 운 좋게 좋은 결과물이 나오기를 기다리는 것이 아니라, 내가 원하는 그림과 소리를 AI로 정교하게 구현해낼 수 있다면 어떨까요?

이번 미션의 목표는 기획 의도가 프롬프트를 거쳐 최종 영상으로 구현되기까지의 전체 제작 파이프라인을 스스로 설계하는 데 있습니다.

가상의(또는 실제) 브랜드 1개를 선택하고, 스토리보드(기획 문서)를 만든 뒤, AI 도구만을 주된 소스로 활용하여 30초 이상 60초 이내의 광고 영상을 완성합니다.

중요한 것은 "도구를 많이 쓰는 것"이 아니라, 기획 의도 → 프롬프트 → 결과물 → 통합 편집으로 이어지는 파이프라인을 스스로 설계하고 왜 그렇게 의사결정했는지 설명하는 것입니다.

또한, 멀티모달 제작에서 빈번히 발생하는 문제(캐릭터/화풍 불일치, 해상도·비율 혼재, 오디오 톤 미스매치, 크레딧/대기열 이슈)를 고려해 현실적인 제작 전략을 세우고 적용합니다.

## 2. 최종 결과물

다음 2가지 산출물을 포함한 AI 기반 브랜드 광고 패키지를 완성한다.

### 스토리보드(기획) 문서 (PDF 1개)

포함 항목:

- 브랜드 아이덴티티
- 캠페인 목표/핵심 메시지
- 씬 구성(씬별 필수 필드)
- 사용 도구 목록

씬별 필수 필드(모든 씬 공통):

- 씬 번호 / 씬 길이(초)
- 목표 메시지(한 문장)
- 화면 구성(구도/피사체/배경/텍스트 유무)
- 내레이션(또는 화면 카피)
- 사용 도구(이미지/비디오/오디오)와 사용 목적
- 입력 프롬프트(원문) + 출력 결과 요약(한 줄)
- 생성 결과 파일명 또는 링크 표기(파일명 규칙 자유, 단 일관성 유지)

추가 권장:

- 최소 1개 씬에 대해 "프롬프트 수정 전/후"와 수정 이유를 기록

### 광고 영상 파일 (MP4 1개)

- 길이: 30초 이상 60초 이내
- 영상/오디오 소스: 생성형 AI 결과물을 주된 소스로 사용(세부 제한은 '제약 사항' 참고)

권장 인코딩 스펙(권장, 필수 아님):

- 해상도: 1080p
- 프레임레이트: 24~30fps
- 비디오 코덱: H.264
- 오디오 코덱: AAC

저사양/제약 허용:

- 해상도 720p 이상(권장 스펙 달성이 어려운 경우)

## 3. 과제 목표

이 과제를 마친 후, 학습자는 아래를 스스로 설명할 수 있어야 한다.

- 텍스트/이미지/비디오/오디오 생성 도구의 역할과 차이(강점/약점)를 말로 설명할 수 있다.
- 기획 의도에 맞는 결과물을 얻기 위해 프롬프트를 어떻게 설계·수정했는지 설명할 수 있다.
- 동일한 요구를 여러 도구로 시도했을 때 결과 차이를 비교하고, 도구 선택의 이유를 설명할 수 있다.
- 생성된 미디어 소스를 해상도/비율/톤앤매너 관점에서 정리하고, 하나의 광고 영상으로 통합하는 흐름을 설명할 수 있다.

## 4. 기능 요구 사항

다음 요구사항을 모두 만족해야 한다.

### 기획(브랜드/캠페인) 정의

- 브랜드(가상 또는 실제) 1개를 선정하고 타겟/톤앤매너/차별점(USP)을 포함한 브랜드 아이덴티티를 정의한다.
- 광고의 목적(인지/전환/리텐션 등)과 핵심 메시지 1개를 한 문장으로 명확히 정의한다.

### 스토리보드 문서화

- 영상 제작 전 씬 단위로 스토리보드를 작성하고 씬별 필수 필드를 누락 없이 채운다.
- 최소 1개 씬에서 프롬프트를 "수정 전/후"로 비교하고 무엇을 왜 바꿨는지(의도/결과)를 기록한다.

### 멀티모달 생성 도구 사용

- 이미지 생성 AI 1종 이상, 비디오 생성(또는 변환) AI 1종 이상, 오디오 생성 AI 1종 이상을 사용한다.
- 도구는 자유롭게 선택하되 각 도구를 "왜 그 목적에 선택했는지"를 스토리보드에 간단히 남긴다.

### 영상 스펙(길이/구성)

- 최종 영상은 30초 이상 60초 이내로 제작한다.
- 영상에는 AI로 생성된 시각 요소(이미지/영상)와 청각 요소(음성/효과음/배경음악 중 1개 이상)가 반드시 포함되어야 한다.

### 서사/메시지(광고 완성도)

단순 컷 나열이 아니라 최소한 다음 중 하나를 만족하는 스토리라인을 구성한다.

- 기승전결(문제 제시 → 전환 → 해결/제안) 구조
- 또는 명확한 메시지 전달("누구에게 어떤 가치를 주는지")이 드러나는 구조

영상의 마지막 3~5초 구간에는 브랜드/제품을 인지할 수 있는 장치(로고, 브랜드명, 슬로건, CTA 중 1개 이상)를 포함한다.

## 5. 보너스 과제 (선택)

### 보너스 1 – 립싱크(Lip-sync) 적용

인물 발화 장면 1개 이상을 추가하고, 입모양과 대사를 자연스럽게 맞춘다.

### 보너스 2 – 동일 스토리보드, 다른 도구로 재제작

이미지 또는 비디오 생성 파트를 다른 도구로 바꿔 동일 씬 1~2개를 재제작한다.

### 보너스 3 – 플랫폼별 화면 비율 버전 제작

동일 스토리보드로 화면 비율 버전 2개 이상을 제작한다.

예:

- 16:9(유튜브)
- 9:16(숏폼/릴스 표준, 1080x1920)
- 1:1(피드용)

## 제약조건

### 6. 제약 사항

### 소스/저작권

- 직접 촬영한 영상, 유료 스톡 영상 소스는 사용하지 않는다.
- 모든 시각 및 청각 소스는 생성형 AI를 통해 만들어진 결과물이어야 한다.
- 저작권·윤리 이슈가 큰 딥페이크, 혐오/선정성/폭력 등 유해 콘텐츠 생성은 금지한다.

### 편집 도구 사용 범위

- 영상 편집 소프트웨어(Premiere Pro, CapCut 등)는 컷 편집, 자막, 간단한 색보정/오디오 레벨 조정 등 "통합 편집" 용도로만 제한적으로 사용한다.
- 영상의 핵심 비주얼/오디오는 AI 생성 결과물을 주된 소스로 유지해야 한다.

### 비용/크레딧

영상 생성은 크레딧 소모가 큰 경우가 많으므로, 이미지 생성 단계에서 스토리보드를 최대한 확정한 뒤 영상 변환을 시도한다.

크레딧이 부족해질 경우, 아래 중 최소 1개 전략을 선택해 범위를 조정한다.

- 씬 수를 줄이고(예: 6씬 → 4씬) 메시지 밀도를 높이기
- 정지 이미지 + 짧은 모션(패닝/줌/루프) 중심으로 구성하기
- 동일한 스타일 레퍼런스를 고정해 재생성 횟수 줄이기

### 도구 접근성 대응

특정 도구가 대기열/플랜 제한으로 막힐 수 있음을 전제로, 이미지/비디오/오디오 각각 대체 도구를 1개 이상 준비한다(도구명만 기록해도 됨).

예:

- 비디오 생성/변환 도구 대안(Pika, Kling 등)
- 오디오 대안(여러 생성 도구 중 택)
- 음성 합성 대안(여러 TTS 중 택)

### 일관성(캐릭터/스타일) 유지 팁(권장)

캐릭터 및 스타일 일관성 유지를 위해, 도구가 제공하는 Character Reference/Style Reference 기능(예: cref, sref) 또는 ControlNet 계열 제어 기능을 적극 활용해본다.

사용했다면, 스토리보드 또는 발표에서 "어떤 파라미터/레퍼런스를 고정했는지"를 설명할 수 있어야 한다.

## Test Case

### 7. 결과 예시

아래는 정답이 아니라 참고 예시다. 실제 문구와 디자인은 달라도 된다.

#### 브랜드 아이덴티티

```markdown
# 브랜드 아이덴티티

- 브랜드명: NeoSeoul
- 타겟: 10~20대(트렌디/스트릿)
- 톤앤매너: 사이버펑크, 네온, 우천의 서울, 시네마틱
- USP: “서울 감성 + 미래적 실루엣”을 결합한 한정 드롭
- 핵심 메시지(한 문장): “오늘 밤, 너의 서울을 입어라.”
```

#### 스토리보드(씬 1 예시)

```markdown
## 씬 1 (Intro)

- 씬 길이(초): 6
- 목표 메시지: “브랜드의 세계관(서울-네온-비)을 한 컷에 각인”
- 화면 구성: 네온사인이 반사되는 젖은 도로 / 인물 실루엣 / 로고는 없음
- 내레이션(또는 카피): “서울의 밤은, 다시 태어난다.”
- 사용 도구 및 목적:
    - 이미지 생성: Midjourney (키 비주얼 생성)
    - 비디오 변환: Runway (이미지를 짧은 모션 컷으로 변환)
    - 오디오: Suno (로파이 비트 + 빗소리)
- 입력 프롬프트(예시): cyberpunk neon Seoul night street, rain reflections, cinematic lighting, 8k, shallow depth of field
- 출력 결과 요약: 네온/우천/도시 질감이 강한 키비주얼 확보
- 결과 파일명(예시): scene01_keyvisual.png / scene01_motion.mp4 / scene01_bgm.wav
```

#### 프롬프트 수정 전/후 기록(최소 1개 씬에 포함 권장)

```markdown
## 씬 2 프롬프트 개선 로그(요약)

- 수정 전 의도: “인물 클로즈업”
- 문제: 얼굴 디테일이 과장되어 브랜드 톤과 불일치
- 수정 후 변경: “인물은 실루엣 중심, 의상 텍스처 강조”로 프롬프트 조정
- 결과 변화: 인물 디테일 과장 감소, 의상/네온 톤 일관성 개선
```
